Coading Approach: Step 1: Set up a new Laravel project.

Step 2: Create a new database in phpMyAdmin named crm.

Step 3: Create models and migration files within the project for the required tables.

Step 4: Define necessary fields in the migration files for proper database structure.

Step 5: Create seeders to populate the database with Admin data.

Step 6: Develop controllers for all modules to handle the business logic.

Step 7: Implement Laravel authentication for admin login.

Step 8: Define routes according to roles in the web.php file for the required endpoints.

Step 9: Develop logic corresponding to the requirements.

Step 10: Test all the functionalities in the Admin panel.

Project setup and run steps
Step 1: Clone the project using the Git repository.

Step 2: Run the Composer command to install dependencies and create the vendor folder, then set up the .env file.

Step 3: Create a new database in phpMyAdmin named is "crm".

Step 4: Run the migration and seeder commands to set up the database schema and insert seed data using the following command. php artisan migrate and php artisan db:seed

Step 5: Run the Project using the following command: php artisan serve
